# Итоговый отчет по лабораторным работам №5 и №6: Техническая документация и построчный анализ кода

Этот документ представляет собой единственное и исчерпывающее руководство по реализации DXF-экспорта. Здесь объединена общая архитектура подсистемы, математическая база и детальный разбор каждой функции модуля.

---

## 1. Общие сведения и архитектура (Лабораторная №5)

**Цель**: Разработка модуля экспорта (DxfExporter), преобразующего внутренние графические примитивы системы в формат DXF (AutoCAD 2000 R15).

### Архитектурные принципы:
Модуль разделен на семь логических этапов записи, соответствующих структуре спецификации DXF:
1.  **HEADER**: Глобальные параметры (версия, кодировка, габариты).
2.  **CLASSES**: Системная информация о структуре данных (пустая секция для AC1015).
3.  **TABLES**: Описание стилей, слоев и типов линий.
4.  **BLOCKS**: Описание "контейнера" для чертежа (Model Space).
5.  **ENTITIES**: Сами графические объекты (Линии, Сплайны и т.д.).
6.  **OBJECTS**: Словари объектов для обеспечения стабильности файла в CAD.
7.  **EOF**: Маркер завершения файла.

## 2. Разбор системной логики (Построчно)

### Конструктор и инициализация
```python
def __init__(self):
    self.handle_seed = 0x10  # Счетчик уникальных ID объектов в HEX
    self.filename = ""       # Текущее имя файла
    self.shapes: List[Any] = [] # Список фигур для экспорта
    self.styles: List[Any] = [] # Список стилей для экспорта
    self.f: Optional[IO] = None # Объект открытого файла (stream)
```
**Комментарий**: Весь DXF построен на ссылках по дескрипторам (Handles). Каждая сущность должна иметь свой ID. Мы начинаем с `0x10`, чтобы не занять адреса системных таблиц (0-F).

### Метод управления записью `_write_pair`
```python
def _write_pair(self, code: int, value: Any) -> None:
    # 10-59: Координаты (X, Y)
    # 140-147: Коэффициенты размеров
    if (10 <= code <= 59) or (140 <= code <= 147):
        # Принудительное форматирование до 6 знаков после запятой
        self.f.write(f"{code}\n{float(value):.6f}\n")
    else:
        # Прямая запись строковых и целых значений
        self.f.write(f"{code}\n{value}\n")
```
**Важно**: Этот метод — основа экспортера. Он гарантирует, что числа не будут записаны в научной нотации (например, 1e-10), которую не понимают многие старые CAD-системы.

## 3. Подсистема управления стилями (Лабораторная №6)

Лабораторная №6 требует переноса атрибутов: цвета, толщины и типов линий.

### Динамическая генерация паттернов
```python
def _get_ltype_pattern(self, style) -> List[float]:
    lt = style.line_type
    dl, gl = style.dash_length, style.gap_length
    if lt == 'dashed':
        return [dl, -gl] # Штрих (положительный), Зазор (отрицательный)
```
**Логика**: Вместо использования зашитых в AutoCAD стилей, мы берем реальные значения `dash_length` из вашего приложения и создаем на их основе нативный паттерн DXF.

### Маппинг слоев (Layer System)
Для сохранения профессиональной структуры чертежа используется иерархия: **Стиль -> Слой -> Объект**.
1.  Каждому уникальному стилю создается свой **Слой** (группа 2 в таблице LAYER).
2.  К слою привязывается **LTYPE** (группа 6).
3.  Назначается цвет (ACI 62 или TrueColor 420).
4.  В итоге все объекты на одном слое наследуют настройки через признак **BYLAYER**.

## 4. Математика геометрических примитивов

Модуль поддерживает 7 обязательных типов фигур плюс дополнительные эмуляции.

### А) Эллипсы (Расчет осей)
В DXF эллипс задается центром и вектором главной оси.
```python
rad = math.radians(shape.rotation)
dx = shape.rx * math.cos(rad) # Координата конца оси по X
dy = shape.rx * math.sin(rad) # Координата конца оси по Y
self._write_pair(11, dx) # Групповой код 11
self._write_pair(21, dy) # Групповой код 21
```

### Б) Волнистые линии (Эмуляция через Arcs)
Чтобы волна была единой линией, мы используем `LWPOLYLINE` с коэффициентом **Bulge**.
*   **Длина полуволны**: `bump_length = length / num_bumps`.
*   **Выпуклость**: `bulge = (2.0 * amplitude) / bump_length`.
*   **Код 42**: Сообщает рендеру DXF, что сегмент между точками А и Б является дугой.

### В) Сплайны (NURBS)
Применяется кубический сплайн (степень 3). При экспорте формируется таблица узлов (Knots), определяющая гладкость кривой. Каждая точка записывается в кодах 10 и 20.

## 5. Административная часть (Handles и Viewport)

### Handle Management
Каждый объект связан с родителем через код `330` (Owner handle). Без этой привязки (например, блока к таблице блоков) файл будет считаться поврежденным.

### Viewport (Центрирование чертежа)
Автоматическое определение центра вида:
```python
cx = (self.ext_min[0] + self.ext_max[0]) / 2.0
cy = (self.ext_min[1] + self.ext_max[1]) / 2.0
# Запись в таблицу VPORT:
self._write_pair(10, cx) # Точка взгляда
self._write_pair(40, height) # Высота картинки (Зум)
```
