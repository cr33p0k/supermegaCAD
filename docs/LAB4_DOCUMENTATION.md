## 1. Иерархия классов геометрических примитивов

### 1.1 Базовый класс `Shape`

**Файл:** `shapes/base.py`

Абстрактный базовый класс определяет единый интерфейс для всех геометрических фигур:

```python
class Shape(ABC):
    """Абстрактный базовый класс для всех фигур"""
    
    def __init__(self):
        self.color: str = "#ff5555"           # Цвет фигуры
        self.selected: bool = False            # Состояние выделения
        self.line_style_name: str = "Сплошная основная"  # Стиль линии
    
    @abstractmethod
    def draw(self, renderer, width, height, view_transform, point_radius=4) -> None:
        """Отрисовка фигуры на canvas"""
    
    @abstractmethod
    def distance_to_point(self, px, py, width, height, view_transform) -> float:
        """Расстояние от точки до фигуры (для выделения)"""
    
    @abstractmethod
    def get_info(self, is_degrees=True) -> str:
        """Информация о фигуре для отображения"""
    
    @abstractmethod
    def get_bounds(self) -> Tuple[float, float, float, float]:
        """Ограничивающий прямоугольник (min_x, min_y, max_x, max_y)"""
    
    @abstractmethod
    def translate(self, dx, dy) -> None:
        """Перемещение фигуры"""
    
    @abstractmethod
    def get_snap_points(self) -> List[Tuple[str, float, float]]:
        """Точки привязок: [('тип', x, y), ...]"""
    
    @abstractmethod
    def get_control_points(self) -> List[Tuple[str, float, float]]:
        """Контрольные точки для редактирования: [('id', x, y), ...]"""
    
    @abstractmethod
    def move_control_point(self, point_id, new_x, new_y) -> None:
        """Переместить контрольную точку"""
    
    @abstractmethod
    def to_dict(self) -> dict:
        """Сериализация в словарь"""
    
    @staticmethod
    @abstractmethod
    def from_dict(data: dict) -> 'Shape':
        """Десериализация из словаря"""
    
    def get_perpendicular_point(self, from_x, from_y) -> Optional[Tuple[float, float]]:
        """Точка перпендикуляра от внешней точки к фигуре"""
    
    def get_tangent_points(self, from_x, from_y) -> List[Tuple[float, float]]:
        """Точки касания от внешней точки к фигуре"""
```

## 2. Реализованные примитивы

### 2.1 Отрезок (Segment)

**Файл:** `shapes/segment.py`

#### Атрибуты:
| Атрибут | Тип | Описание |
|---------|-----|----------|
| `x1, y1` | float | Координаты начальной точки |
| `x2, y2` | float | Координаты конечной точки |

#### Способы создания:
| Режим | Описание | Клавиша M для переключения |
|-------|----------|---------------------------|
| Две точки | Начальная точка → конечная точка | ✓ |
| Длина и угол | Начальная точка → ввод длины и угла | ✓ |

#### Точки привязок:
- `endpoint` — начало и конец отрезка
- `midpoint` — середина отрезка

#### Контрольные точки:
- `start` — начальная точка
- `end` — конечная точка
- `midpoint` — середина (перемещает весь отрезок)

#### Редактирование через панель свойств:
- X₁, Y₁ — координаты начала
- X₂, Y₂ — координаты конца
- Вычисляемые: длина, угол наклона

💡 **Пояснение для преподавателя:**

**Базовая фигура:** Отрезок — простейший примитив, основа для всех остальных. Реализация тривиальна: две точки и прямая между ними.

**Особенности реализации:**
- **Расстояние до точки:** Используется проекция точки на прямую через скалярное произведение. Если проекция попадает на отрезок (t ∈ [0,1]) — расстояние до проекции, иначе — до ближайшего конца.
- **Перпендикуляр:** Тот же алгоритм проекции — если проекция на отрезке, это и есть перпендикуляр.
- **Стили линий:** Поддержка волнистых и ломаных линий через генерацию промежуточных точек.

**Почему важно:** Отрезок используется как базовый элемент для аппроксимации сложных фигур (многоугольники, сплайны) при вычислении пересечений и привязок.

### 2.2 Окружность (Circle)

**Файл:** `shapes/circle.py`

#### Атрибуты:
| Атрибут | Тип | Описание |
|---------|-----|----------|
| `cx, cy` | float | Координаты центра |
| `radius` | float | Радиус |

#### Способы создания:
| Режим | Описание | Точек |
|-------|----------|-------|
| **Центр и радиус** | Центр → точка на окружности | 2 |
| **Центр и диаметр** | Центр → точка (радиус = расстояние/2) | 2 |
| **Две точки** | Диаметр по двум противоположным точкам | 2 |
| **Три точки** | Окружность через три произвольные точки | 3 |

#### Методы создания:
```python
@classmethod
def from_two_points(cls, x1, y1, x2, y2) -> 'Circle':
    """Создание по двум точкам (диаметр)"""

@classmethod  
def from_three_points(cls, x1, y1, x2, y2, x3, y3) -> Optional['Circle']:
    """Создание по трём точкам на окружности"""
```

#### Точки привязок:
- `center` — центр окружности
- `quadrant` — 4 квадрантные точки (верх, низ, лево, право)

#### Контрольные точки:
- `center` — центр (перемещает окружность)
- `radius` — точка на окружности (изменяет радиус)

#### Редактирование через панель свойств:
- Центр X, Центр Y — координаты центра
- Радиус — радиус окружности
- Вычисляемые: диаметр, длина окружности, площадь

💡 **Пояснение для преподавателя:**

**Математическая основа:** Окружность задаётся уравнением (x-cx)² + (y-cy)² = r². Все вычисления основаны на этом уравнении.

**Создание по трём точкам:**
- Решается система из 3 уравнений (подстановка точек в уравнение окружности)
- Используется метод определителей Крамера для нахождения центра
- Радиус = расстояние от центра до любой из точек

**Квадрантные точки:** 4 точки на окружности (0°, 90°, 180°, 270°) от центра. Используются для привязок — стандартная функция в САПР.

**Отрисовка:** Через `canvas.create_oval()` или полилинию из точек. Полилиния используется для поддержки стилей линий (пунктир, волнистая).

**Почему важно:** Окружность — базовая кривая. Дуга, эллипс, многоугольник так или иначе связаны с окружностью.

### 2.3 Дуга (Arc)

**Файл:** `shapes/arc.py`

#### Атрибуты:
| Атрибут | Тип | Описание |
|---------|-----|----------|
| `cx, cy` | float | Координаты центра |
| `radius` | float | Радиус |
| `start_angle` | float | Начальный угол (градусы) |
| `end_angle` | float | Конечный угол (градусы) |

#### Способы создания:
| Режим | Описание | Точек |
|-------|----------|-------|
| **Три точки** | Начало → конец → контрольная точка (выпуклость) | 3 |
| **Центр и углы** | Центр → начальный угол → конечный угол | 3 |

#### Методы создания:
```python
@classmethod
def from_three_points(cls, x1, y1, x2, y2, x3, y3) -> Optional['Arc']:
    """Создание дуги по трём точкам:
    x1, y1 - начало дуги
    x2, y2 - конец дуги
    x3, y3 - точка на дуге, определяющая её положение и выпуклость.
    Все три точки лежат на одной окружности.
    Автоматически выбирает направление (CW/CCW) так, чтобы дуга проходила через третью точку."""
```

#### Точки привязок:
- `center` — центр дуги
- `endpoint` — начало и конец дуги
- `midpoint` — середина дуги

#### Контрольные точки:
- `center` — центр дуги
- `start` — начальная точка (меняет начальный угол)
- `end` — конечная точка (меняет конечный угол)

#### Редактирование через панель свойств:
- Центр X, Центр Y — координаты центра
- Радиус — радиус дуги
- Начальный угол, Конечный угол — углы в градусах
- Вычисляемые: длина дуги

💡 **Пояснение для преподавателя:**

**Сложность работы с углами:** Главная проблема дуги — корректная обработка углов, особенно когда дуга проходит через 0° или больше 180°.

**Нормализация углов:**
- Углы хранятся в градусах, могут быть любыми (в т.ч. отрицательными)
- Метод `_get_extent()` правильно вычисляет угловой размах с учётом направления
- Если дуга создана с extent > 180°, это сохраняется при отрисовке

**Создание по трём точкам:**
- Все три точки лежат на одной окружности
- Первая точка определяет начало дуги
- Вторая точка определяет конец дуги
- Третья точка лежит на дуге и определяет её положение и выпуклость:
  - Находится окружность, проходящая через все три точки (используется формула для центра окружности через три точки)
  - Определяется направление дуги (CW/CCW) так, чтобы при движении от первой точки ко второй дуга проходила через третью точку
  - Радиус и центр окружности определяются однозначно через три точки
  - Третья точка определяет, какая из двух возможных дуг (короткая или длинная) будет построена

**Переключение направления (клавиша N):**
- Позволяет выбрать короткую (60°) или длинную (300°) дугу между двумя углами
- Реализовано через изменение extent при сохранении начального угла

**Отрисовка:** Полилиния из точек, вычисленных по параметрическому уравнению: `x = cx + r*cos(θ), y = cy + r*sin(θ)`, где θ меняется от start_angle до end_angle.

**Почему важно:** Дуга — часто используемый элемент в технических чертежах (скругления, арки, сегменты).

### 2.4 Прямоугольник (Rectangle)

**Файл:** `shapes/rectangle.py`

#### Атрибуты:
| Атрибут | Тип | Описание |
|---------|-----|----------|
| `x, y` | float | Координаты левого нижнего угла |
| `width` | float | Ширина |
| `height` | float | Высота |
| `corner_radius` | float | Радиус скругления углов |
| `chamfer` | float | Размер фаски |
| `rotation` | float | Угол поворота (градусы) |

#### Способы создания:
| Режим | Описание | Точек |
|-------|----------|-------|
| **Две точки** | Противоположные углы прямоугольника | 2 |
| **Точка и размеры** | Угол + ввод ширины и высоты | 2 |
| **Центр и размеры** | Центр + половина ширины/высоты | 2 |

#### Методы создания:
```python
@classmethod
def from_two_points(cls, x1, y1, x2, y2, 
                    corner_radius=0, chamfer=0, rotation=0) -> 'Rectangle':
    """Создание по двум противоположным точкам"""

@classmethod
def from_center(cls, cx, cy, width, height, 
                corner_radius=0, chamfer=0, rotation=0) -> 'Rectangle':
    """Создание по центру и размерам"""
```

#### Модификаторы углов:
- **Скругление углов** (`corner_radius`) — создаёт дуговые скругления на углах
- **Фаски** (`chamfer`) — создаёт срез углов под 45°

**Особенность:** При наличии скругления или фаски точки привязок располагаются на концах дуг/фасок, а не на геометрических углах прямоугольника.

💡 **Пояснение для преподавателя (детальное):**

**Проблема:** Когда у прямоугольника есть скругления или фаски, геометрические углы (координаты x, y) не совпадают с видимыми точками контура. Если бы привязки были к геометрическим углам, пользователь не смог бы точно привязаться к реальному контуру фигуры.

**Решение:** Метод `get_corner_feature_points()` вычисляет реальные точки контура:
- Если нет модификаторов → возвращает 4 угла (как обычно)
- Если есть скругление/фаска → возвращает 8 точек (по 2 на каждый угол: точка входа и выхода дуги/фаски)

**Реализация:**
```python
def get_corner_feature_points(self):
    corners = self.get_corners()  # Углы с учётом поворота
    offset = corner_r if corner_r > 0 else chamfer_size
    
    for i, (cx, cy) in enumerate(corners):
        prev = corners[(i - 1) % 4]  # Предыдущий угол
        nxt = corners[(i + 1) % 4]   # Следующий угол
        
        # Векторы от угла к соседним углам
        v_prev = normalize(prev - corner, offset)
        v_next = normalize(nxt - corner, offset)
        
        # Точки входа и выхода
        entry = corner + v_prev
        exit_ = corner + v_next
```

**Почему это важно:** В реальных САПР (AutoCAD, SolidWorks) привязки всегда к видимым элементам. Это позволяет точно позиционировать новые объекты относительно существующих, что критично для технических чертежей.

**Пример использования:** При создании линии от конца скругления к другому объекту пользователь видит точку привязки именно на конце дуги, а не где-то в невидимом углу.

#### Точки привязок:
- `center` — центр прямоугольника
- `endpoint` — углы или концы скруглений/фасок (8 точек при наличии модификаторов)
- `midpoint` — середины 4 сторон

#### Контрольные точки:
- `corner_bl, corner_br, corner_tr, corner_tl` — четыре угла
- `center` — центр

#### Редактирование через панель свойств:
- X, Y — координаты левого нижнего угла
- Ширина, Высота — размеры
- **Поворот** — угол поворота в градусах *(дополнительная функция)*
- Скругление — радиус скругления углов
- Фаска — размер фаски
- Вычисляемые: периметр, площадь, диагональ

💡 **Пояснение для преподавателя (детальное):**

**Зачем нужен поворот:** В задании не требовался поворот прямоугольника, но это стандартная функция в САПР. Позволяет создавать наклонные элементы без необходимости строить их из отрезков.

**Реализация через матрицу поворота:**
```python
def _rotate_point(self, px, py, cx, cy):
    """Поворот точки вокруг центра на угол rotation"""
    if self.rotation == 0:
        return (px, py)
    
    rad = math.radians(self.rotation)
    cos_r, sin_r = math.cos(rad), math.sin(rad)
    
    # Смещаем в начало координат
    dx, dy = px - cx, py - cy
    
    # Применяем матрицу поворота
    # [cos -sin] [dx]
    # [sin  cos] [dy]
    new_x = cx + dx * cos_r - dy * sin_r
    new_y = cy + dx * sin_r + dy * cos_r
    
    return (new_x, new_y)
```

**Что поворачивается:**
- Все углы прямоугольника
- Все точки привязок (концы скруглений/фасок, середины сторон)
- Контрольные точки для редактирования

**Границы (bounds):** Вычисляются как минимальный ограничивающий прямоугольник вокруг всех повёрнутых углов:
```python
def get_bounds(self):
    corners = self.get_corners()  # Уже повёрнутые
    xs = [c[0] for c in corners]
    ys = [c[1] for c in corners]
    return (min(xs), min(ys), max(xs), max(ys))
```

**Интеграция с редактированием:** При изменении угла поворота в панели свойств прямоугольник мгновенно перерисовывается. При перетаскивании контрольных точек поворот сохраняется.

**Почему это важно:** В реальных чертежах часто нужны наклонные элементы (рамки, панели под углом). Без поворота пришлось бы строить их из отрезков, что неудобно.

### 2.5 Эллипс (Ellipse)

**Файл:** `shapes/ellipse.py`

#### Атрибуты:
| Атрибут | Тип | Описание |
|---------|-----|----------|
| `cx, cy` | float | Координаты центра |
| `rx` | float | Большая полуось |
| `ry` | float | Малая полуось |
| `rotation` | float | Угол поворота (градусы) |

#### Способы создания:
| Режим | Описание | Точек |
|-------|----------|-------|
| **Центр и оси** | Центр → конец большой оси → конец малой оси | 3 |
| **Две точки** | Ограничивающий прямоугольник (bounding box) | 2 |

#### Методы создания:
```python
@classmethod
def from_center_and_axes(cls, cx, cy, axis1_x, axis1_y, 
                          axis2_x, axis2_y) -> 'Ellipse':
    """Создание по центру и конечным точкам осей.
    Автоматически вычисляет угол поворота."""
```

#### Точки привязок:
- `center` — центр эллипса
- `quadrant` — 4 точки на концах осей

#### Контрольные точки:
- `center` — центр эллипса
- `axis_x` — конец большой полуоси (меняет rx и rotation)
- `axis_y` — конец малой полуоси (меняет ry)

#### Редактирование через панель свойств:
- Центр X, Центр Y — координаты центра
- Полуось X, Полуось Y — размеры полуосей
- Поворот — угол поворота в градусах
- Вычисляемые: периметр (приближённо), площадь

💡 **Пояснение для преподавателя:**

**Параметрическое уравнение:** Эллипс задаётся как `x = rx*cos(θ), y = ry*sin(θ)` с последующим поворотом на угол `rotation`.

**Поворот через матрицу:**
```python
# Поворот точки на эллипсе
rot = math.radians(self.rotation)
x_rot = x * cos(rot) - y * sin(rot) + cx
y_rot = x * sin(rot) + y * cos(rot) + cy
```

**Создание по осям:**
- Первая точка задаёт большую полуось (rx) и угол поворота
- Вторая точка задаёт малую полуось (ry)
- Автоматически вычисляется, какая ось больше

**Аппроксимация для привязок:**
- Эллипс аппроксимируется 48 сегментами для вычисления пересечений и перпендикуляров
- Точность достаточна для визуального отображения и привязок (ошибка < 0.1%)

**Квадрантные точки:** 4 точки на концах осей (с учётом поворота). Используются для привязок.

**Почему важно:** Эллипс — обобщение окружности, часто используется в технических чертежах (овалы, эллиптические отверстия).

### 2.6 Многоугольник (Polygon)

**Файл:** `shapes/polygon.py`

#### Атрибуты:
| Атрибут | Тип | Описание |
|---------|-----|----------|
| `cx, cy` | float | Координаты центра |
| `radius` | float | Радиус (вписанной или описанной окружности) |

|-------|----------|-------|
| **Центр и радиус** | Центр → точка на окружности (задаёт радиус и начальный поворот) | 2 |

#### Варианты построения:
- **Вписанный** — вершины лежат на окружности заданного радиуса
- **Описанный** — стороны касаются окружности заданного радиуса

#### Точки привязок:
- `center` — центр многоугольника
- `endpoint` — все вершины
- `midpoint` — середины всех сторон

#### Контрольные точки:
- `center` — центр многоугольника
- `radius` — первая вершина (меняет радиус и поворот)

#### Редактирование через панель свойств:
- Центр X, Центр Y — координаты центра
- Радиус — радиус окружности
- Сторон — количество сторон (spinbox 3-100)
- Поворот — угол поворота в градусах
- Вписанный — чекбокс выбора типа построения
- Вычисляемые: длина стороны, апофема, периметр, площадь

💡 **Пояснение для преподавателя:**

**Геометрия правильного многоугольника:**
- Все стороны равны, все углы равны
- Вершины лежат на окружности
- Формула вершины: `(cx + r*cos(α), cy + r*sin(α))`, где α = rotation + i*360°/n

**Вписанный vs Описанный:**
- **Вписанный:** Вершины на окружности радиуса r
- **Описанный:** Стороны касаются окружности радиуса r
  - Эффективный радиус = r / cos(π/n) для описанного
  - Апофема (расстояние от центра до стороны) = r для описанного

**Вычисление параметров:**
- Сторона: `2 * r * sin(π/n)`
- Апофема: `r * cos(π/n)` для вписанного, `r` для описанного
- Площадь: `(n * сторона * апофема) / 2`

**Отрисовка:** Полилиния, соединяющая вершины. Многоугольник уже состоит из отрезков, поэтому аппроксимация для привязок не нужна.

**Почему важно:** Правильные многоугольники часто используются в технических чертежах (гайки, болты, шестерни, декоративные элементы).

**Сплайн Катмулла-Рома:**
- Интерполяционный сплайн, проходящий через все контрольные точки
- Параметр `tension` (0-1) контролирует "жёсткость" кривой
  - 0.0 = очень жёсткая (близко к ломаной)
  - 1.0 = очень плавная (сильно изгибается)

- Формула Катмулла-Рома вычисляет промежуточные точки с учётом соседей

**Особенности:**
- Кривая всегда проходит через контрольные точки (в отличие от B-spline)
- Не требует дополнительных контрольных точек (в отличие от B-spline)
- Проще в реализации, чем NURBS

**Аппроксимация для привязок:**
- Сплайн аппроксимируется 12 точками на сегмент
- Это даёт достаточную точность для визуального отображения и привязок

**Редактирование:**
- Можно добавлять, удалять и перемещать контрольные точки
- При перемещении точки кривая автоматически пересчитывается

**Почему важно:** Сплайны используются для создания плавных кривых произвольной формы (профили, контуры, траектории).

### 3.5 Алгоритм поиска привязок

💡 **Пояснение для преподавателя (детальное):**

**Проблема выбора привязки:** В радиусе захвата может быть несколько точек привязок. Какую выбрать? Самую близкую? Но тогда привязка к сетке может перебить более важную привязку к концу объекта.

**Решение — система приоритетов:**
```python
PRIORITIES = {
    SnapType.ENDPOINT: 1,      # Самый важный
    SnapType.MIDPOINT: 2,
    SnapType.CENTER: 2,
    SnapType.INTERSECTION: 3,
    SnapType.QUADRANT: 4,
    SnapType.PERPENDICULAR: 5,
    SnapType.TANGENT: 5,
    SnapType.GRID: 20          # Менее важный
}
```

**Алгоритм выбора:**
```python
best, best_d = None, float('inf')
for p in pts:
    sx, sy = world_to_screen(p.x, p.y)
    d = distance(cursor, p)  # Расстояние в пикселях
    
    # Взвешенное расстояние = расстояние + штраф за приоритет
    weighted_d = d + p.priority * 0.1
    
    if d <= SNAP_RADIUS and weighted_d < best_d:
        best_d = weighted_d
        best = p
```

**Пример работы:**
- Курсор на расстоянии 10px от конца объекта (приоритет 1)
- Курсор на расстоянии 8px от узла сетки (приоритет 20)
- Взвешенные расстояния:
  - Конец: 10 + 1*0.1 = 10.1
  - Сетка: 8 + 20*0.1 = 10.0
- Выбирается сетка (10.0 < 10.1)

Но если конец на расстоянии 7px:
- Конец: 7 + 1*0.1 = 7.1
- Сетка: 8 + 20*0.1 = 10.0
- Выбирается конец (7.1 < 10.0)

**Почему коэффициент 0.1:** Это означает, что разница в приоритете 10 эквивалентна 1 пикселю расстояния. Таким образом, привязка к концу (приоритет 1) будет выбрана, даже если сетка (приоритет 20) ближе на 1-2 пикселя.

**Альтернативные подходы:**
1. **Только расстояние** — не учитывает важность привязок
2. **Жёсткие приоритеты** — всегда выбирать по приоритету, игнорируя расстояние — слишком жёстко
3. **Взвешенное расстояние** (наш подход) — баланс между расстоянием и важностью

**Вопрос на защите:** "Почему именно коэффициент 0.1?"
**Ответ:** Подобран эмпирически для комфортной работы. Можно было бы сделать настраиваемым, но для базовой реализации фиксированное значение достаточно. В профессиональных САПР обычно тоже фиксированные приоритеты.

Основной метод `find_snap_point()` выполняет следующие шаги:

```python
def find_snap_point(self, cx, cy, shapes, w, h, vt, from_pt=None):
    # 1. Преобразование экранных координат в мировые
    wx, wy = vt.screen_to_world(cx, cy, w, h)
    pts = []
    
    # 2. Сбор точек от всех фигур
    for s in shapes:
        pts.extend(self._shape_snaps(s, wx, wy, from_pt))
    
    # 3. Поиск пересечений между парами фигур
    if self._enabled.get(SnapType.INTERSECTION):
        for ix, iy in self._all_intersections(shapes):
            pts.append(SnapPoint(SnapType.INTERSECTION, ix, iy, ...))
    
    # 4. Привязка к сетке
    if self._enabled.get(SnapType.GRID):
        gx = round(wx / self._grid_step) * self._grid_step
        gy = round(wy / self._grid_step) * self._grid_step
        pts.append(SnapPoint(SnapType.GRID, gx, gy, ...))
    
    # 5. Выбор лучшей точки с учётом расстояния и приоритета
    best, best_d = None, float('inf')
    for p in pts:
        sx, sy = vt.world_to_screen(p.x, p.y, w, h)
        d = math.hypot(cx - sx, cy - sy)
        # Расстояние + штраф за низкий приоритет
        if d <= SNAP_RADIUS and d + p.priority * 0.1 < best_d:
            best_d = d + p.priority * 0.1
            best = p
    
    return best
```

### 3.6 Алгоритмы вычисления пересечений

💡 **Пояснение для преподавателя (детальное):**

**Зачем нужны пересечения:** Позволяют точно позиционировать объекты в точках пересечения существующих фигур. Это критично для технических чертежей, где часто нужно строить от точек пересечения линий.

**Подходы к реализации:**

**1. Точные аналитические решения** (для простых фигур):

**Отрезок–отрезок:** Параметрические уравнения
- Отрезок 1: P(t) = P1 + t*(P2-P1), t ∈ [0,1]
- Отрезок 2: Q(u) = Q1 + u*(Q2-Q1), u ∈ [0,1]
- Решаем систему: P(t) = Q(u)
- Проверяем: 0 ≤ t ≤ 1 и 0 ≤ u ≤ 1

**Отрезок–окружность:** Квадратное уравнение
- Подставляем параметрическое уравнение отрезка в уравнение окружности
- Получаем квадратное уравнение относительно t
- Решаем через дискриминант
- Проверяем: 0 ≤ t ≤ 1

**Окружность–окружность:** Геометрическое решение
- Вычисляем расстояние между центрами d
- Проверяем условия существования пересечения: |r1-r2| ≤ d ≤ r1+r2
- Находим точку на линии между центрами: a = (r1² - r2² + d²) / (2d)
- Вычисляем высоту: h = √(r1² - a²)
- Две точки пересечения симметричны относительно линии центров

**2. Аппроксимация сегментами** (для сложных фигур):

**Почему аппроксимация:**
- Эллипс: нет простого аналитического решения для пересечения эллипса с другими фигурами
- Сплайн: кривая Безье требует численных методов
- Многоугольник: можно решать аналитически, но проще через сегменты

**Как работает:**
```python
# Эллипс → 48 сегментов
pts = [ellipse.get_point_on_ellipse(i * 360/48) for i in range(48)]
segs = [(pts[i], pts[i+1]) for i in range(48)]

# Ищем пересечения каждого сегмента с целевой фигурой
for seg in segs:
    intersections.extend(find_intersection(seg, target_shape))
```

**Точность аппроксимации:**
- Эллипс: 48 сегментов → ошибка < 0.1% для типичных размеров
- Сплайн: 12 точек на сегмент → достаточная точность для визуального отображения

**3. Комбинация для дуг:**

```python
# Сначала решаем для базовой окружности
circle_intersections = circle_circle_intersection(arc.circle, other_shape)

# Затем фильтруем по углам
for p in circle_intersections:
    angle = atan2(p.y - arc.cy, p.x - arc.cx)
    if angle_on_arc(angle, arc.start_angle, arc.end_angle):
        result.append(p)
```

**Устранение дублей:**
```python

for p in all_intersections:
    if not any(distance(p, u) < 0.01 for u in unique):
        unique.append(p)
```
Допуск 0.01 единиц — точки считаются одинаковыми, если ближе этого расстояния.

**Производительность:**
- Для N фигур проверяем N*(N-1)/2 пар
- Для каждой пары — O(1) для простых фигур, O(k) для аппроксимации (k — количество сегментов)
- Общая сложность: O(N² * k) в худшем случае
- Для типичных чертежей (10-50 фигур) это достаточно быстро

**Вопрос на защите:** "Почему не использовать библиотеки для геометрических вычислений?"
**Ответ:** Для учебной работы важно показать понимание алгоритмов. Использование готовых библиотек скрывает детали реализации. В реальных проектах, конечно, можно использовать библиотеки типа Shapely, но здесь мы демонстрируем знание математики и алгоритмов.

#### Отрезок–отрезок (параметрическое уравнение):

```python
def _seg_seg(self, x1, y1, x2, y2, x3, y3, x4, y4):
    """Пересечение двух отрезков методом параметрических уравнений"""
    d = (x1 - x2) * (y3 - y4) - (y1 - y2) * (x3 - x4)
    if abs(d) < 1e-10: return None  # Параллельны
    
    t = ((x1 - x3) * (y3 - y4) - (y1 - y3) * (x3 - x4)) / d
    u = -((x1 - x2) * (y1 - y3) - (y1 - y2) * (x1 - x3)) / d
    
    # Проверка принадлежности точки обоим отрезкам
    if 0 <= t <= 1 and 0 <= u <= 1:
        return (x1 + t * (x2 - x1), y1 + t * (y2 - y1))
    return None
```

#### Отрезок–окружность (квадратное уравнение):

```python
def _seg_circ(self, x1, y1, x2, y2, cx, cy, r):
    """Пересечение отрезка и окружности"""
    dx, dy = x2 - x1, y2 - y1
    fx, fy = x1 - cx, y1 - cy
    
    a = dx*dx + dy*dy
    b = 2 * (fx*dx + fy*dy)
    c = fx*fx + fy*fy - r*r
    
    disc = b*b - 4*a*c
    if disc < 0: return []  # Нет пересечений
    
    sd = math.sqrt(disc)
    # Возвращаем точки, где t ∈ [0, 1]
    return [(x1 + t*dx, y1 + t*dy) 
            for t in [(-b - sd)/(2*a), (-b + sd)/(2*a)] 
            if 0 <= t <= 1]
```

#### Окружность–окружность (геометрическое решение):

```python
def _circ_circ(self, x1, y1, r1, x2, y2, r2):
    """Пересечение двух окружностей"""
    d = math.hypot(x2 - x1, y2 - y1)
    
    # Проверка существования пересечения
    if d > r1 + r2 or d < abs(r1 - r2) or d == 0:
        return []
    
    # Расстояние от центра 1 до линии пересечения
    a = (r1*r1 - r2*r2 + d*d) / (2*d)
    h2 = r1*r1 - a*a
    if h2 < 0: return []
    h = math.sqrt(h2)
    
    # Точка на линии между центрами
    px = x1 + a * (x2 - x1) / d
    py = y1 + a * (y2 - y1) / d
    
    if h < 0.001:  # Касание
        return [(px, py)]
    
    # Две точки пересечения
    return [
        (px + h * (y2 - y1) / d, py - h * (x2 - x1) / d),
        (px - h * (y2 - y1) / d, py + h * (x2 - x1) / d)
    ]
```

#### Дуги — фильтрация по углам:

```python
def _on_arc(self, p, arc) -> bool:
    """Проверка принадлежности точки дуге по углу"""
    angle = math.degrees(math.atan2(p[1] - arc.cy, p[0] - arc.cx))
    start = arc.start_angle
    extent = arc.end_angle - arc.start_angle
    
    # Корректная обработка углов > 180°
    if extent >= 0:
        rel = (angle - start) % 360
        return rel <= extent
    else:
        rel = -((start - angle) % 360)
        return rel >= extent
```

#### Сложные фигуры — аппроксимация сегментами:

💡 **Пояснение для преподавателя (детальное):**

**Проблема:** Не все фигуры можно описать простыми уравнениями:
- **Эллипс:** Уравнение (x/a)² + (y/b)² = 1, но пересечение с другими фигурами требует сложных вычислений
- **Сплайн:** Кривая Безье или NURBS — нет простых аналитических решений
- **Многоугольник:** Можно решать аналитически, но проще через сегменты

**Решение — аппроксимация сегментами:**

**Принцип:** Любую кривую можно приблизить ломаной линией (набором отрезков). Чем больше сегментов, тем выше точность, но ниже производительность.

**Для эллипса:**
```python
# 48 сегментов = 360° / 48 = 7.5° на сегмент
pts = [ellipse.get_point_on_ellipse(i * 360/48) for i in range(48)]
segs = [(pts[i], pts[(i+1)%48]) for i in range(48)]
```

**Точность:** Максимальная ошибка аппроксимации эллипса сегментами:
- Для 48 сегментов: ошибка < 0.1% для типичных размеров
- Для визуального отображения и привязок этого более чем достаточно

**Для сплайна:**
```python
# 12 точек на сегмент сплайна
pts = spline.get_curve_points(12)
segs = [(pts[i], pts[i+1]) for i in range(len(pts)-1)]
```

**Почему 12 точек:** Сплайн может иметь сложную форму. 12 точек на сегмент обеспечивает плавную аппроксимацию даже для изогнутых участков.

**Для многоугольника:**
```python
# Многоугольник уже состоит из отрезков
vertices = polygon.get_vertices()
segs = [(vertices[i], vertices[(i+1)%len(vertices)]) 
        for i in range(len(vertices))]
```

**Почему не аналитически для многоугольника:** Можно было бы решать пересечения аналитически, но использование единого подхода (через сегменты) упрощает код и делает его более универсальным.

**Производительность:**
- Для эллипса: 48 сегментов → 48 проверок пересечений
- Для сплайна: зависит от количества контрольных точек
- Для многоугольника: количество сегментов = количество сторон

**Оптимизация:** Можно было бы использовать пространственные индексы (R-tree) для ускорения поиска пересечений, но для учебной работы это избыточно.

**Вопрос на защите:** "Почему именно 48 сегментов для эллипса?"
**Ответ:** 48 — компромисс между точностью и производительностью. Можно было бы использовать 24 (быстрее, но менее точно) или 96 (точнее, но медленнее). 48 даёт ошибку < 0.1%, что незаметно визуально и достаточно для привязок. В профессиональных САПР тоже используются подобные значения (обычно 32-64 сегмента).

```python
def _get_segs(self, shape) -> List[Tuple[float, float, float, float]]:
    """Аппроксимация фигуры набором отрезков"""
    t = type(shape).__name__
    
    if t == 'Rectangle':
        c = shape.get_corners()
        return [(c[i][0], c[i][1], c[(i+1)%4][0], c[(i+1)%4][1]) 
                for i in range(4)]
    
    if t == 'Polygon':
        v = shape.get_vertices()
        return [(v[i][0], v[i][1], v[(i+1)%len(v)][0], v[(i+1)%len(v)][1]) 
                for i in range(len(v))]
    
    if t == 'Ellipse':
        # 48 сегментов для аппроксимации эллипса
        pts = [shape.get_point_on_ellipse(i * 360/48) for i in range(48)]
        return [(pts[i][0], pts[i][1], pts[(i+1)%48][0], pts[(i+1)%48][1]) 
                for i in range(48)]
    
    if t == 'Spline':
        pts = shape.get_curve_points(12)  # 12 точек на сегмент
        return [(pts[i][0], pts[i][1], pts[i+1][0], pts[i+1][1]) 
                for i in range(len(pts)-1)]
    
    return []
```

### 3.7 Алгоритм перпендикуляра

💡 **Пояснение для преподавателя (детальное):**

**Что такое перпендикуляр:** Точка на фигуре, такая что линия от from_point к этой точке перпендикулярна касательной к фигуре в этой точке.

**Для отрезка:**
```python
# Проекция точки на прямую
dx, dy = x2 - x1, y2 - y1
len2 = dx*dx + dy*dy
t = ((fx - x1)*dx + (fy - y1)*dy) / len2

# Проверяем, что проекция на отрезке
if 0 <= t <= 1:
    return (x1 + t*dx, y1 + t*dy)
```

**Математика:** Скалярное произведение вектора от начала отрезка к from_point на направляющий вектор отрезка даёт параметр t. Если t ∈ [0,1], проекция на отрезке.

**Для окружности:**
```python
# Точка на радиусе через from_point
d = distance(from_point, center)
if d < 0.001: return None  # Точка в центре — нет перпендикуляра

# Нормализуем вектор от центра к from_point и умножаем на радиус
return (center + radius * (from_point - center) / d)
```

**Почему это работает:** Для окружности перпендикуляр всегда лежит на радиусе, проходящем через from_point. Это следует из того, что радиус перпендикулярен касательной.

**Для дуги:** То же самое, что для окружности, но с проверкой принадлежности точки дуге:
```python
p = circle_perpendicular(...)
if angle_on_arc(p, arc.start_angle, arc.end_angle):
    return p
```

**Для сложных фигур (прямоугольник, многоугольник):**
```python
segs = get_segments(shape)
best, best_d = None, float('inf')

for seg in segs:
    p = perpendicular_to_segment(seg, from_point)
    if p:
        d = distance(from_point, p)
        if d < best_d:
            best, best_d = p, d

return best
```

**Почему перебор сегментов:** Для многоугольника перпендикуляр может быть на любой стороне. Перебираем все стороны и выбираем ближайший перпендикуляр.

**Вопрос на защите:** "Почему для эллипса не используется аналитическое решение?"
**Ответ:** Для эллипса перпендикуляр требует решения системы нелинейных уравнений, что сложно. Аппроксимация через сегменты даёт достаточную точность (ошибка < 1%) и проще в реализации. В профессиональных САПР тоже часто используется дискретизация для сложных кривых.

```python
def _perp(self, s, fx, fy) -> Optional[Tuple[float, float]]:
    """Найти точку перпендикуляра от (fx, fy) к фигуре"""
    t = type(s).__name__
    
    if t == 'Segment':
        # Проекция точки на прямую
        dx, dy = s.x2 - s.x1, s.y2 - s.y1

        t = ((fx - s.x1)*dx + (fy - s.y1)*dy) / len2

            return (s.x1 + t*dx, s.y1 + t*dy)
        return None
    
    if t == 'Circle':
        # Точка на радиусе через from_point
        d = math.hypot(fx - s.cx, fy - s.cy)
        if d < 0.001: return None
        return (s.cx + s.radius*(fx - s.cx)/d, 
                s.cy + s.radius*(fy - s.cy)/d)
    
    if t == 'Arc':
        # Как для окружности + проверка принадлежности дуге
        ...
    
    # Для сложных фигур — перебор всех сегментов
    segs = self._get_segs(s)
    best, best_d = None, float('inf')
    for seg in segs:
        p = self._perp_seg(*seg, fx, fy)
        if p:
            d = math.hypot(p[0] - fx, p[1] - fy)

```

### 3.8 Алгоритм касательной

💡 **Пояснение для преподавателя (детальное):**

**Что такое касательная:** Точка на кривой, такая что линия от from_point к этой точке касается кривой (перпендикулярна радиусу в этой точке).

**Для окружности (геометрическое решение):**

**Математика:**
- Из from_point к центру окружности: расстояние d
- Радиус окружности: r
- Угол между радиусом и касательной: α = arccos(r/d)
- Базовый угол от центра к from_point: β = atan2(fy - cy, fx - cx)

**Две точки касания:**
```python
angle = math.acos(r / d)  # Угол между радиусом и касательной
base = math.atan2(fy - cy, fx - cx)  # Направление к from_point

# Две точки симметричны относительно радиуса
pt1 = (cx + r*cos(base + angle + π/2), cy + r*sin(base + angle + π/2))
pt2 = (cx + r*cos(base - angle - π/2), cy + r*sin(base - angle - π/2))
```

**Почему π/2:** Касательная перпендикулярна радиусу, поэтому добавляем 90°.

**Проверка существования:** Если d ≤ r (from_point внутри или на окружности), касательных нет.

**Для дуги:** То же решение, но фильтруем точки по углам:
```python
pts = circle_tangent_points(...)
return [p for p in pts if angle_on_arc(p, arc.start_angle, arc.end_angle)]
```

**Для эллипса (дискретизация):**

**Проблема:** Нет простого аналитического решения для касательных к эллипсу.

**Решение:** Дискретизация контура и поиск точек, где касательная ⊥ радиус-вектору.

```python
# 72 точки на контуре эллипса
pts = [ellipse.get_point_on_ellipse(i * 5) for i in range(72)]

for i, p in enumerate(pts):
    # Вектор касательной (разность соседних точек)
    pp, pn = pts[(i-1) % 72], pts[(i+1) % 72]
    tangent = (pn[0] - pp[0], pn[1] - pp[1])
    
    # Вектор к from_point
    to_from = (fx - p[0], fy - p[1])
    
    # Проверка перпендикулярности (cos ≈ 0)
    cos_angle = dot_product(tangent, to_from) / (|tangent| * |to_from|)
    if abs(cos_angle) < 0.1:  # ~84°-96°
        result.append(p)
```

**Почему 0.1:** Это допуск для перпендикулярности. cos(84°) ≈ 0.1, cos(96°) ≈ -0.1. Учитывая дискретизацию, это разумный допуск.

**Почему 72 точки:** 360° / 72 = 5° на точку. Это даёт точность ~1-2% для типичных размеров эллипсов.

**Альтернативный подход:** Можно было бы использовать численные методы (например, метод Ньютона) для точного решения, но это сложнее и медленнее. Дискретизация даёт достаточную точность для визуального отображения.

**Вопрос на защите:** "Почему для эллипса не используется аналитическое решение?"
**Ответ:** Аналитическое решение касательных к эллипсу требует решения системы нелинейных уравнений, что выходит за рамки базовой геометрии. Дискретизация — стандартный подход в вычислительной геометрии для сложных кривых. Точность 72 точек достаточна для практических целей (ошибка < 2%).

```python
def _tangent(self, s, fx, fy) -> List[Tuple[float, float]]:
    """Найти точки касания от (fx, fy) к фигуре"""
    t = type(s).__name__
    
    if t in ('Circle', 'Arc'):
        cx, cy, r = s.cx, s.cy, s.radius
        d = math.hypot(fx - cx, fy - cy)
        if d <= r: return []  # Точка внутри — нет касательных
        
        # Угол между радиусом и касательной
        angle = math.acos(r / d)
        base = math.atan2(fy - cy, fx - cx)
        
        pts = [
            (cx + r*math.cos(base + angle + math.pi/2),
             cy + r*math.sin(base + angle + math.pi/2)),
            (cx + r*math.cos(base - angle - math.pi/2),
             cy + r*math.sin(base - angle - math.pi/2))
        ]
        
        # Для дуги — фильтрация по углам
        if t == 'Arc':
            pts = [p for p in pts if self._on_arc(p, s)]
        return pts
    
    if t == 'Ellipse':
        # Дискретизация: поиск точек где касательная ⊥ радиус-вектору
        pts = [s.get_point_on_ellipse(i * 5) for i in range(72)]
        result = []
        for i, p in enumerate(pts):
            # Вектор касательной (разность соседних точек)
            pp, pn = pts[(i-1) % 72], pts[(i+1) % 72]
            tx, ty = pn[0] - pp[0], pn[1] - pp[1]
            # Вектор к from_point
            vx, vy = fx - p[0], fy - p[1]
            # Проверка перпендикулярности (cos ≈ 0)
            tl, vl = math.hypot(tx, ty), math.hypot(vx, vy)
            if tl > 0 and vl > 0:
                cos_angle = abs(tx*vx + ty*vy) / (tl * vl)
                if cos_angle < 0.1:  # ~84°-96°
                    result.append(p)
        return result[:2]
    
    return []
```

### 3.9 Привязка к сетке

```python
# Вычисление ближайшего узла сетки
if self._enabled.get(SnapType.GRID):
    gx = round(wx / self._grid_step) * self._grid_step
    gy = round(wy / self._grid_step) * self._grid_step
    pts.append(SnapPoint(SnapType.GRID, gx, gy, priority=20))
```

Шаг сетки синхронизируется с настройками отображения сетки в приложении через метод `set_grid_step()`.

💡 **Пояснение для преподавателя (детальное):**

**Зачем нужна привязка к сетке:** Позволяет создавать объекты с точными координатами, кратными шагу сетки. Особенно полезно для технических чертежей, где важна точность и выравнивание.

**Алгоритм:**
```python
# Вычисление ближайшего узла сетки
gx = round(wx / self._grid_step) * self._grid_step
gy = round(wy / self._grid_step) * self._grid_step
```

**Как это работает:**
1. `wx / self._grid_step` — делим координату на шаг (например, 50 / 25 = 2.0)
2. `round(...)` — округляем до целого (2.0 → 2)
3. `* self._grid_step` — умножаем обратно (2 * 25 = 50)

**Пример:** Если курсор на (47, 53) и шаг сетки 25:
- gx = round(47/25) * 25 = round(1.88) * 25 = 2 * 25 = 50
- gy = round(53/25) * 25 = round(2.12) * 25 = 2 * 25 = 50
- Результат: привязка к (50, 50)

**Синхронизация с визуальной сеткой:**
```python
# В main.py при изменении шага сетки
def _on_grid_step_changed(self):
    step = float(self.grid_step_var.get())
    self.snap_manager.set_grid_step(step)  # Обновляем шаг привязки
    self.redraw()  # Перерисовываем сетку
```

**Почему синхронизация важна:** Если визуальная сетка показывает узлы каждые 25 единиц, а привязка работает с шагом 50, пользователь будет путаться — видит один узел, а привязывается к другому.

**Приоритет 20 (самый низкий):**
- Конец объекта (приоритет 1) всегда выиграет, даже если сетка ближе
- Это правильно, т.к. привязка к объектам важнее привязки к сетке
- Сетка используется только когда нет других привязок в радиусе

**Визуальная индикация:** Маленький крестик `+` размером 4 пикселя, чтобы не перекрывать другие элементы.

**Вопрос на защите:** "Почему не сделали привязку к сетке обязательной?"
**Ответ:** Сетка — вспомогательный инструмент. В некоторых случаях она мешает (например, при работе с объектами, которые не выровнены по сетке). Поэтому она выключена по умолчанию и имеет низкий приоритет.

### 3.10 Визуальная индикация

💡 **Пояснение для преподавателя (детальное):**

**Зачем нужна визуальная индикация:** Без индикации пользователь не знает, к какой точке он привязывается. Это может привести к ошибкам — например, привязаться к центру вместо конца.

**Цветовая кодировка:**
- **Зелёный** (конец) — самый важный, используется чаще всего
- **Голубой** (середина) — важная, но менее частая
- **Пурпурный** (центр) — легко заметен, не путается с другими
- **Оранжевый** (пересечение) — предупреждающий цвет, т.к. пересечений может быть много
- **Жёлтый** (квадрант) — яркий, хорошо виден
- **Серый** (сетка) — нейтральный, не отвлекает

**Символы:**
- **□ Квадрат** (конец) — простая форма, легко распознаётся
- **△ Треугольник** (середина) — указывает на середину
- **○ Круг** (центр) — символ центра
- **✕ Крестик** (пересечение) — стандартный символ пересечения
- **⊥ Перпендикуляр** — математический символ
- **◎ Касательная** — круг с линией
- **+ Крестик** (сетка) — маленький, не перекрывает другие элементы

**Размеры символов:**
- Обычные: 10 пикселей (sz = 10)
- Сетка: 4 пикселя (компактный)
- Касательная: 5 пикселей (круг) + линия

**Подпись типа:**
```python
canvas.create_text(sx+sz+5, sy-sz-5, 
                  text=self.NAMES[s.snap_type], 
                  fill=c, anchor="sw")
```
Текст отображается рядом с символом, чтобы пользователь точно знал тип привязки.

**Обновление индикатора:**
- Индикатор перерисовывается на каждом движении мыши
- Используется тег "snap_indicator" для быстрого удаления старого
- Рисуется поверх всех объектов (z-order)

**Соответствие стандартам:** Цвета и символы соответствуют стандартам AutoCAD, что делает интерфейс знакомым для пользователей профессиональных САПР.

**Вопрос на защите:** "Почему не использовать только цвета или только символы?"
**Ответ:** Комбинация цвета и символа обеспечивает максимальную узнаваемость. Цвета помогают быстро различать типы, символы дают точную информацию даже при проблемах с цветовосприятием. Это стандартный подход в доступности интерфейсов.

Каждый тип привязки отображается уникальным символом:

```python
def draw_snap_indicator(self, canvas, w, h, vt):

    sx, sy = vt.world_to_screen(s.x, s.y, w, h)

    elif s.snap_type == SnapType.MIDPOINT:

    elif s.snap_type == SnapType.CENTER:
        canvas.create_oval(sx-sz, sy-sz, sx+sz, sy+sz, outline=c)
    elif s.snap_type == SnapType.INTERSECTION:
        canvas.create_line(sx-sz, sy-sz, sx+sz, sy+sz, fill=c)
        canvas.create_line(sx-sz, sy+sz, sx+sz, sy-sz, fill=c)
    elif s.snap_type == SnapType.GRID:
        canvas.create_line(sx-4, sy, sx+4, sy, fill=c)
        canvas.create_line(sx, sy-4, sx, sy+4, fill=c)
    # ... и т.д.
    
    # Подпись типа привязки
    canvas.create_text(sx+sz+5, sy-sz-5, text=self.NAMES[s.snap_type], fill=c)
```

### 3.11 Управление привязками

В правой панели интерфейса доступны чекбоксы для включения/выключения каждого типа привязок:

```
○ Центр        ◇ Квадрант
✕ Пересеч.     ⊥ Перпенд.

```

По умолчанию включены все привязки кроме сетки.

💡 **Пояснение для преподавателя (детальное):**

**Интеграция привязок — ключевая особенность:**

**1. При создании фигур (`DrawTool`):**
```python
def _get_snap_point(self, event):
    px, py = event.x, event.y
    from_point = self._points[0] if self._points else None
    
    snap = self.app.snap_manager.find_snap_point(
        px, py, shapes, w, h, view_transform, from_point
    )
    if snap:
        return (snap.x, snap.y)  # Используем точку привязки
    return screen_to_world(px, py)  # Иначе — позиция курсора
```

**Особенность:** При наличии первой точки (`from_point`) система автоматически ищет перпендикуляры и касательные, что позволяет строить объекты под прямым углом или касательно к существующим.

**2. При редактировании контрольных точек:**
```python
# В select_tool.py и edit_tool.py
def on_mouse_move(self, event):
    if self._dragging_point:
        world_point = self._get_snap_point(event)  # ← Использует привязки!
        shape.move_control_point(point_id, world_point[0], world_point[1])
```

**Почему это важно:** При перетаскивании угла прямоугольника можно привязать его к концу другого объекта, что обеспечивает точное выравнивание.

**3. При перемещении фигур:**
```python
# При перетаскивании всей фигуры привязки не используются напрямую,
# но можно было бы добавить привязку центра фигуры к другим объектам
```

**Управление привязками:**

**Глобальное включение/выключение:**
```python
snap_manager.set_enabled(True/False)
```
Отключает всю систему привязок одним кликом.

**Выборочное управление:**
```python
snap_manager.set_snap_type_enabled(SnapType.GRID, False)
```
Можно отключить конкретный тип, если он мешает.

**Примеры использования:**
- **Отключить сетку** — если нужно привязываться только к объектам
- **Отключить пересечения** — если их слишком много и они мешают
- **Отключить касательные** — если работаешь только с прямыми линиями

**Вопрос на защите:** "Почему не сделать привязки всегда включенными?"
**Ответ:** В некоторых случаях привязки мешают — например, при создании объектов, которые не должны быть привязаны к существующим. Возможность отключения даёт гибкость. В профессиональных САПР тоже есть возможность управления привязками.

**Производительность:**
- Поиск привязок вызывается на каждом движении мыши
- Для оптимизации можно было бы использовать пространственные индексы
- Для учебной работы текущая реализация достаточна (работает плавно для 50-100 фигур)

**Решение:** Вызов `properties_panel.refresh()` в методе `on_mouse_move()` при перетаскивании.

**Реализация:**
```python

        shape, point_id = self._dragging_point
        world_point = self._get_snap_point(event)
        shape.move_control_point(point_id, world_point[0], world_point[1])

        if hasattr(self.app, 'properties_panel'):
            self.app.properties_panel.refresh()  # ← Ключевой момент
        
        self.app.redraw()
```

**Метод refresh():**
```python
def refresh(self):
    """Обновить значения полей без пересоздания виджетов"""
    self._updating = True  # Флаг предотвращает рекурсию
    try:
        for attr, var in self._vars.items():
            if hasattr(self.current_shape, attr):
                value = getattr(self.current_shape, attr)
                if isinstance(var, tk.DoubleVar):
                    var.set(float(value))  # Обновляем значение
    finally:
        self._updating = False
```

**Защита от рекурсии:** Флаг `_updating` предотвращает бесконечный цикл: изменение в панели → обновление фигуры → обновление панели → ...

**Преимущества:**
1. **Обратная связь в реальном времени** — пользователь видит точные значения сразу
2. **Точное позиционирование** — можно ориентироваться на числа, а не только на визуал
3. **Профессиональный UX** — как в AutoCAD, SolidWorks

**Пример использования:** Перетаскиваешь угол прямоугольника и видишь, как меняются X, Y, ширина, высота в панели. Можешь остановиться, когда значения станут нужными.

💡 **Пояснение для преподавателя (детальное):**

**Проблема без автоматического снятия:** Пользователь выделил фигуру для редактирования, затем решил нарисовать новую. Если выделение не снимается, при клике на canvas может случайно переместиться старая фигура вместо создания новой — это путает.

**Решение:** В методе `set_tool()` при переключении на "draw" автоматически снимаем выделение.

**Реализация:**
```python
def set_tool(self, tool_name: str):
    self.current_tool.on_deactivate()
    if tool_name in self.tools:
        self.current_tool, label = self.tools[tool_name]
        self.tool_label.config(text=label)
    self.current_tool.on_activate()
    
    # При переключении на рисование снимаем выделение
    if tool_name == "draw":
        self.shape_manager.deselect_all()
        self.properties_panel.update_for_shape(None)  # Очищаем панель
    
    self.redraw()
```

**Почему это важно:**
1. **Чёткое разделение режимов** — режим рисования ≠ режим редактирования
2. **Предотвращение ошибок** — нельзя случайно переместить объект при создании нового
3. **Интуитивность** — поведение соответствует ожиданиям пользователя

**Альтернативный подход (не реализован):** Можно было бы оставить выделение, но тогда нужно было бы проверять в `DrawTool`, не кликнул ли пользователь по выделенной фигуре. Это усложняет код и может привести к багам.

**Сравнение с профессиональными САПР:** В AutoCAD при переключении инструмента выделение обычно снимается, что подтверждает правильность подхода.

## 7. Дополнительно: Числовой ввод при построении

**Файл:** `tools/draw_tool.py`

💡 **Пояснение для преподавателя (детальное):**

**Зачем нужен числовой ввод:** В задании не требовался, но это стандартная функция всех САПР (AutoCAD, SolidWorks, Inventor). Позволяет создавать объекты с точными размерами без "угадывания" координат мышью.

**Архитектура:**
```python
class DrawTool:
    def __init__(self):
        self._input_values = ["", ""]      # Буферы для двух полей
        self._input_field_index = 0         # Активное поле (0 или 1)
        self._input_active = False
```

**Как это работает:**
1. Пользователь начинает вводить цифры → они попадают в буфер `_input_values[field_index]`
2. Панель ввода отображается в правом нижнем углу с текущими значениями
3. При Enter или клике → вызывается `_apply_numeric_input()`, который вычисляет точку из введённых значений
4. Точка добавляется в `_points` и используется для создания фигуры

**Пример для прямоугольника:**
```python
# Пользователь ввёл: ширина=100, высота=50
# Курсор на (200, 150), первая точка на (100, 100)

width = 100  # из ввода
height = 50  # из ввода
cx, cy = 100, 100  # первая точка

# Определяем направление от курсора
dir_x = 1 if 200 >= 100 else -1  # → 1
dir_y = 1 if 150 >= 100 else -1  # → 1

# Вычисляем вторую точку
pt2 = (cx + width*dir_x, cy + height*dir_y)  # (200, 150)
```

**Гибкость — комбинирование ввода и мыши:**
- Можно ввести только ширину, высоту задать мышью
- Можно ввести оба значения
- Можно не вводить ничего — работает как обычно

**Почему два поля:** Некоторые режимы требуют два значения (X и Y, ширина и высота, длина и угол). Переключение между полями — Tab.

**Визуальная панель:**
- Показывает текущие значения (серым), если ничего не введено
- Показывает введённые значения (белым), если есть ввод
- Подсвечивает активное поле
- Показывает курсор в активном поле

**Преимущества:**
1. **Точность** — можно задать размеры с точностью до 0.01
2. **Скорость** — быстрее, чем позиционировать мышью
3. **Повторяемость** — можно создать несколько одинаковых объектов

**Вопрос на защите:** "Почему не сделали отдельное диалоговое окно?"
**Ответ:** Панель ввода не перекрывает рабочую область и всегда видна. Диалоговое окно нужно было бы открывать/закрывать, что замедляет работу. Это стандартный подход в современных САПР.

### 7.1 Описание функционала

При создании любого примитива доступна панель числового ввода, позволяющая вводить точные значения координат, размеров или углов вместо использования мыши.

💡 **Пояснение для преподавателя (детальное):**

**Управление вводом — интуитивные горячие клавиши:**

**Tab — переключение полей:**
```python

    fields = self._get_input_fields_config()
    if len(fields) > 1:
        self._input_field_index = (self._input_field_index + 1) % len(fields)
```
Циклическое переключение между полями позволяет быстро вводить несколько значений.

```python
    if self._has_any_input():
```

Enter применяет введённые значения и создаёт точку. Если точек достаточно — создаётся фигура.

```python
if key == 'Escape':

        self._reset_state()  # Отменить построение
```
Двухуровневая отмена: сначала очищает ввод, затем отменяет построение.

**Автоматический возврат фокуса:**

1. **Гибкость** — не обязательно вводить все значения
2. **Скорость** — быстрее, чем вводить всё численно
3. **Точность** — можно точно задать одно значение, другое — визуально

**Вопрос на защите:** "Почему не использовать диалоговое окно для ввода?"
**Ответ:** Панель ввода не перекрывает рабочую область и всегда видна. Диалоговое окно нужно открывать/закрывать, что замедляет работу. Это стандартный подход в современных САПР (AutoCAD, Fusion 360). Панель ввода позволяет видеть текущие значения и вводить новые одновременно.

### 7.5 Особенности работы

1. **Комбинирование ввода и курсора:**
   - Если введено только одно значение, второе берётся от положения курсора
   - Например, для прямоугольника можно ввести только ширину, а высоту задать мышью

2. **Направление от курсора:**
   - При вводе размеров направление (знак) определяется положением курсора относительно начальной точки
   - Это позволяет контролировать, в какую сторону строится фигура

3. **Привязки при вводе:**
   - Числовой ввод работает совместно с системой привязок
   - После применения ввода следующая точка снова использует привязки

💡 **Пояснение для преподавателя (детальное):**

**Гибкость системы ввода — три режима работы:**

**1. Полностью числовой ввод:**
```
Пользователь: вводит X=100, Y=50, ширина=200, высота=100
Результат: Точный прямоугольник с заданными параметрами
```
Используется для технических чертежей, где важна точность.

**2. Комбинированный ввод:**
```
Пользователь: вводит ширину=200, высоту задаёт мышью
Результат: Ширина точная, высота визуальная
```
Используется когда нужно точно задать один параметр, другой — приблизительно.

**3. Полностью визуальный:**
```
Пользователь: не вводит ничего, только кликает мышью
Результат: Работает как обычно, без числового ввода
```
Используется для быстрых набросков.

**Направление от курсора:**

**Проблема:** При вводе размеров (ширина, высота) нужно знать, в какую сторону строить фигуру.

**Решение:**
```python
# Определяем направление от курсора
dir_x = 1 if cursor_x >= start_x else -1
dir_y = 1 if cursor_y >= start_y else -1

# Применяем направление к размерам
pt2 = (start_x + width * dir_x, start_y + height * dir_y)
```

**Пример:**
- Стартовая точка: (100, 100)
- Введено: ширина=50, высота=30
- Курсор на: (150, 130) → dir_x=1, dir_y=1
- Результат: (150, 130) — прямоугольник вправо-вверх

Если курсор на (50, 70) → dir_x=-1, dir_y=-1 → прямоугольник влево-вниз

**Почему это важно:** Без учёта направления курсора прямоугольник всегда строился бы в одну сторону (например, вправо-вверх), что неудобно.

**Обработка частичного ввода:**

**Примеры комбинирования:**

**Прямоугольник:**
- Введено только ширина → высота от курсора
- Введено только высота → ширина от курсора

**Окружность (центр и радиус):**
- Введён радиус → направление от курсора
- Не введён радиус → радиус от курсора

**Вопрос на защите:** "Почему не требовать ввод всех значений?"
**Ответ:** Гибкость важнее строгости. Пользователь может работать в разных режимах в зависимости от задачи. Для точных чертежей — вводит всё численно, для быстрых набросков — использует мышь. Это стандартный подход в профессиональных САПР.
